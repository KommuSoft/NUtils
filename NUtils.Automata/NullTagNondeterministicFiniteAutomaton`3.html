NullTagNondeterministicFiniteAutomaton<TStateTag,TEdgeTag,TCollection> Generic Class

An implementation of the INullTagNondeterministicFiniteAutomaton`2 interface representing a nondeterministic finite automaton that has a definition for a special TEdgeTag that represents the "epsilon"-transition: a transition that doesn't eat any item of the sequence.

Syntax

public class NullTagNondeterministicFiniteAutomaton<TStateTag, TEdgeTag, TCollection> : NUtils.NondeterministicFiniteAutomaton<TStateTag, TEdgeTag, TCollection>, NUtils.Abstract.ICloneable<INondeterministicFiniteAutomaton<TStateTag, TEdgeTag>>, NUtils.Abstract.ICloneable<INullTagNondeterministicFiniteAutomaton<TStateTag, TEdgeTag>>, INondeterministicFiniteAutomaton<TStateTag, TEdgeTag>, INullTagNondeterministicFiniteAutomaton<TStateTag, TEdgeTag>, ICloneable
where TCollection : ICollection<NUtils.Automata.IState<TStateTag, TEdgeTag>>, new()

Type Parameters

TStateTag
The type of the tags that are assigned to the nodes.
TEdgeTag
The type of the tags that are assigned to the edges.
TCollection
The type of collection used to store different states given they share the same tag.

Remarks

Most implementations of this interface will require that the tags are unique per state: two states can't share the same tag.

Parameters

edges
The list of Tuple`3 instances representing edges to be added to the new NullTagNondeterministicFiniteAutomaton`3.
initialStateTag
The tag of the state that is the initial state.
acceptingStateTags
The list of tags representing the accepting states.
nullEdge
The tag associated with the epsilon edge: an edge that doesn't "eat" any item in the verifying sequence.

Remarks

Edges with tags that are not associated with any state will result in the creation of new IState`2 instances in this NullTagNondeterministicFiniteAutomaton`3.

If the given initialStateTag does not correspond with any registered state, a IState`2 instance will be created.

For each tag in the list of acceptingStateTags, the "first" IState`2 that matches the tag will be picked, if no IState`2 instance matches the given tag, the tag is ignored.

initialStateTag
The tag of the state that is the initial state.
acceptingStateTags
The list of tags representing the accepting states.
nullEdge
The tag associated with the epsilon edge: an edge that doesn't "eat" any item in the verifying sequence.

Remarks

If the given list of states is not effective, then no states are added.

Non-effective states will not be added to the list of states.

Edges with tags that are not associated with any state will result in the creation of new IState`2 instances in this NullTagNondeterministicFiniteAutomaton`3.

If the given initialStateTag does not correspond with any registered state, a IState`2 instance will be created.

For each tag in the list of acceptingStateTags, the "first" IState`2 that matches the tag will be picked, if no IState`2 instance matches the given tag, the tag is ignored.
